Pipeline Method

Our approach turns unstructured web pages into structured records with transparent quality controls. The system is designed to be maintainable under real-world conditions (layout changes, duplicates, noise).

1) Collect

Scheduler triggers per-source collection jobs (hourly/daily/weekly).

- Rate limiting + backoff
- Cache (ETag/Last-Modified when available)
- Store raw HTML for reproducibility
2) Parse & Extract

Extract structured fields with source-specific rules, then map to a unified schema.

Fields: title, published_at, category, url, source, summary, tags
Fallbacks: regex date extraction + heuristic category classifier
3) Deduplicate

Reduce repeated items caused by mirrored pages or minor edits.

- Exact match: URL
- Near-duplicate: title similarity
- Content fingerprint: hash of cleaned text
4) Score & Alert

Confidence score is based on completeness and stability, and alerts are generated when needed.

score = completeness(0-60) + parser_health(0-30) + source_trust(0-10)
alert triggers: missing key fields | DOM diff over threshold | repeated failures
Assessment (how we evaluated)

We assessed outcomes using both technical and user-facing measures:

- Field completeness rate (%)
- Duplicate reduction (%)
- False alert rate (manual review sample)
- Time-to-find: user test on “latest policy update” task
