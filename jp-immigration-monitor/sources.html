Source Strategy

We prioritize official, publicly available channels. Each source is evaluated by: update frequency, content importance, accessibility, and scraping permissions. If scraping is disallowed, we use “link-only” curation instead.

Source Inventory (template)
Type Source Content Suggested Frequency Structured Fields
Official Notices National / agency announcements Policy changes, procedural updates Daily title, date, category, url, summary
FAQ / Guidance Help pages How-to steps, requirements, forms Weekly topic, last_updated, section, url
Regional Offices Local office pages Office hours, holiday schedules Weekly / seasonal region, notice, date, url
Change Monitor DOM diff + checks Layout changes affecting parsers Every run source_id, diff_score, alert_level
Normalization Rules

We normalize across sources to support consistent search and analytics:

category: Notice | Policy | Alert | Guidance
published_at: ISO-8601 UTC
language: en / ja (optional)
confidence: 0-100 (based on field completeness + parser stability)
